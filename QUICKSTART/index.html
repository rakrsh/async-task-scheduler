    Quick Start Guide¶

    Get up and running with the Async Task Scheduler in minutes!

    🚀 One-Command Setup¶

    Choose your platform:

    Windows (PowerShell)¶

    .\setup.ps1

    Or use the batch launcher: 

    launch.bat setup

    macOS / Linux (Bash)¶

    chmod +x setup.sh
./setup.sh

    Cross-Platform (Node.js)¶

    node setup.js

    📋 Available Commands¶

    Full Setup (Recommended First Time)¶

    Installs all dependencies and builds the project.

    PowerShell: 

    .\setup.ps1                    # Default: full setup
.\setup.ps1 -Mode setup        # Explicit setup mode

    Bash: 

    ./setup.sh                     # Default: full setup
./setup.sh help                # Show help

    Node.js: 

    node setup.js                  # Default: full setup
node setup.js --help           # Show help

    Batch (Windows): 

    launch.bat setup
launch.bat help

    Development Mode¶

    Runs full setup then starts the application with hot-reload.

    PowerShell: 

    .\setup.ps1 -Mode dev

    Bash: 

    ./setup.sh dev

    Node.js: 

    node setup.js --dev

    Batch: 

    launch.bat dev

    Build Only¶

    Builds the C++ project without running anything.

    PowerShell: 

    .\setup.ps1 -Mode build

    Bash: 

    ./setup.sh build

    Node.js: 

    node setup.js --build

    Batch: 

    launch.bat build

    Run Already-Built Project¶

    Starts the dashboard and API server (requires prior build).

    PowerShell: 

    .\setup.ps1 -Mode run

    Bash: 

    ./setup.sh run

    Node.js: 

    node setup.js --run

    Batch: 

    launch.bat run

    Clean Build Artifacts¶

    Removes all build directories and node_modules.

    PowerShell: 

    .\setup.ps1 -Mode clean

    Bash: 

    ./setup.sh clean

    Node.js: 

    node setup.js --clean

    Batch: 

    launch.bat clean

    🎯 What Happens During Setup?¶

    The setup process performs these steps in order:

    1. Check Prerequisites
    2. CMake (3.20+)
    3. C++ Compiler (MSVC 19.28+, GCC 10+, or Clang 11+)
    4. Node.js (14+)

    5. vcpkg (optional, will be cloned if needed)

    6. Install C++ Dependencies

    7. Initializes vcpkg if not present

    8. Installs dependencies listed in vcpkg.json

    9. Install Node.js Dependencies

    10. Root package.json dependencies
    11. Server dependencies (ui/server/package.json)

    12. Client dependencies (ui/client/package.json)

    13. Build C++ Project

    14. Generates CMake build files with vcpkg toolchain
    15. Compiles scheduler, tests, and benchmarks

    16. Outputs binaries to build/Release/

    17. Ready to Run

    18. Dashboard available at http://localhost:3000
    19. API server at http://localhost:3001

    🌐 Accessing the Application¶

    After setup completes, open your browser:

    • Dashboard: http://localhost:3000
    • Real-time task visualization

    • Worker metrics and performance graphs

    • API Documentation: http://localhost:3001/api/metrics

    • REST endpoints for metrics

    • Main Documentation: http://localhost:8000 (run mkdocs serve in another terminal)

    • Architecture, guides, and API reference

    🔧 Prerequisites¶

    Windows¶

    • Visual Studio 2019+ or Build Tools for C++
    • CMake 3.20+
    • Node.js 14+

    macOS¶

    # Using Homebrew:
brew install cmake node

    Linux (Ubuntu/Debian)¶

    sudo apt-get install cmake build-essential nodejs npm

    🆘 Troubleshooting¶

    CMake not found¶

    Windows: Install from cmake.org 

    # Or via Chocolatey:
choco install cmake

    macOS: 

    brew install cmake

    Linux: 

    sudo apt-get install cmake

    C++ compiler not found¶

    Windows: Install Visual Studio Community

    macOS: 

    xcode-select --install

    Linux (GCC): 

    sudo apt-get install build-essential

    Linux (Clang): 

    sudo apt-get install clang

    Node.js not found¶

    Install from nodejs.org or:

    macOS: 

    brew install node

    Linux (Ubuntu/Debian): 

    curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

    vcpkg bootstrap fails¶

    The setup script handles this automatically. If you encounter issues:

    # Windows:
.\vcpkg\bootstrap-vcpkg.bat

# macOS/Linux:
./vcpkg/bootstrap-vcpkg.sh

    Port already in use¶

    If ports 3000 or 3001 are already in use, you can change them:

    Dashboard client: Edit ui/client/src/components/Dashboard.js and modify REACT_APP_API_BASE

    Server port: Edit ui/server/.env and change PORT=3001 to another port

    Build fails with linking errors¶

    # Clean and rebuild from scratch:
.\setup.ps1 -Mode clean  # or ./setup.sh clean
.\setup.ps1 -Mode build  # or ./setup.sh build

    Dashboard shows "Connection Error"¶

    Ensure the Express server is running: 

    # Check if server is listening:
curl http://localhost:3001/api/health

    If not running, restart: 

    .\setup.ps1 -Mode run  # or ./setup.sh run

    🎓 Understanding the Structure¶

    async-task-scheduler/
├── setup.ps1              # Windows PowerShell setup
├── setup.sh               # macOS/Linux Bash setup
├── setup.js               # Cross-platform Node.js setup
├── launch.bat             # Windows batch launcher
├── CMakeLists.txt         # C++ build configuration
├── vcpkg.json             # C++ dependency manifest
├── ui/                    # Web dashboard (Node.js + React)
│   ├── server/            # Express API server
│   ├── client/            # React dashboard
│   └── package.json       # NPM workspace config
├── src/                   # C++ source code
├── include/scheduler/     # C++ headers
├── tests/                 # Unit tests
├── benchmarks/            # Performance benchmarks
└── docs/                  # Documentation

    💡 Tips¶

    • Speed Up Builds: The setup uses parallel compilation (--parallel). More cores = faster builds.
    • Skip Setup: If you've already built once, use setup.ps1 -Mode run to skip the build.
    • Clean Slate: Use setup.ps1 -Mode clean to remove all build artifacts before troubleshooting.
    • Verbose Builds: Remove --silent flags in setup scripts for detailed build output.
    • Check Logs: Look for errors in the terminal output; they usually indicate missing dependencies.
